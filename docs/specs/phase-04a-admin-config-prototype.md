# Phase 04A 后台配置维护台原型开发文档

## 1. 原型目标

本原型用于生成和开发 `/admin-config` 后台配置维护台。它不是移动 H5 页面，而是面向 HRBP、产品运营、权限 Owner、内容 Owner 的桌面网页端工作台。

后台配置台的目标是让运营人员完成 P0 配置闭环：

- 配置岗位权限包。
- 配置 D1 引导入口。
- 配置新人首周反馈表。
- 配置匿名反馈三级联动。
- 管理知识库资料元数据。
- 处理匿名反馈池。

所有 P0 数据必须通过真实后端 API 和数据库保存。保存后刷新页面仍要保留。新人端、复盘端必须读取后台配置后的数据。

## 2. 产品边界

必须保留：

- 不做真实登录，后台操作者先固定为 `demo-admin`。
- 不调用真实飞书 API。
- 不调用真实审批 API。
- 不调用真实 RAG、LLM、向量库。
- 不做真实文件上传解析，知识库只保存元数据，解析和向量化状态为模拟。
- 不物理删除配置，优先使用 `enabled` 启停。
- 不向管理者端暴露匿名反馈原文。

后台配置台是 page 08，路由仍是：

- `/admin-config`
- `/admin-config?tab=knowledge`
- `/admin-config?tab=feedback`

不要新增独立页面破坏 12 页最终页面地图。所有后台能力都收敛在 page 08 内，通过左侧导航、顶部 tab、表格、抽屉或弹窗切换。

## 3. 数据边界

### 配置表

这些数据由后台配置台维护，影响后续展示：

- `roles`：岗位。
- `permission_items`：权限项、Owner、申请入口、理由模板、常见等待原因。
- `role_permission_items`：岗位与权限项关系。
- `d1_guide_configs`：D1 引导入口。
- `weekly_feedback_questions`：首周反馈问题。
- `weekly_feedback_options`：首周反馈选项。
- `anonymous_feedback_modules`：匿名反馈模块。
- `anonymous_feedback_problem_types`：匿名反馈问题类型。
- `anonymous_feedback_expected_actions`：匿名反馈期望处理方式。
- `knowledge_base_docs`：知识库元数据。

### 业务记录表

这些数据是新人、反馈、管理者实际操作记录，后台不能随意清空：

- `permission_progress`
- `follow_up_tasks`
- `weekly_feedbacks`
- `anonymous_feedbacks`
- `manager_feedback_actions`

### 统计派生数据

复盘页指标从配置表和业务记录表计算，不在后台手工填写：

- 新人数。
- 权限提交数。
- 4 小时回访待处理数。
- 匿名反馈数。
- 首周反馈数。
- 知识库资料数。

  - 岗位权限包

  - 知识库管理

顶部栏：

- 标题：后台配置维护台。

  - 当前操作者：demo-admin
  - 最近保存时间
  - “刷新数据”按钮

主内容区：

- 上方为 3 到 4 个指标卡：
  - 岗位数
  - 权限项数
  - 知识库资料数

### 交互模式

统一采用：

- 表格用于浏览。
- 右侧抽屉用于编辑。
- 保存按钮固定在抽屉底部。
- 取消按钮关闭抽屉并丢弃未保存变更。
- 启用/停用使用 switch，不出现删除按钮。
- 保存成功 toast：`已保存配置` 或模块专属文案。
- 保存失败显示字段级错误和顶部错误提示。

## 6. 模块二：岗位权限包管理

### 用途

配置某个岗位的必开权限、可选权限、Owner、申请入口、理由模板、常见等待原因。

### 表格字段

权限项表格：

- 权限名称
- 所属分类
- 类型：必开 / 可选
- Owner
- 申请入口
- 审批人
- 常见等待原因数量
- 状态：启用 / 停用
- updatedBy
- updatedAt
- 操作：编辑

岗位选择区：

- 当前岗位：协同办公产品实习生。
- 岗位部门。
- 岗位描述。
- 权限包完整度。

### 右侧编辑抽屉

字段：

- 权限名称，必填。
- 所属分类，必填。
- 权限类型，单选：必开 / 可选。
- Owner，必填。
- Owner 联系方式，必填。
- 申请入口，必填，允许 `mock-feishu://approval/...` 或 http/https。
- 审批人，必填。
- 理由模板，多行文本，必填。
- 常见等待原因，多行文本，一行一个。
- 启用状态 switch。

底部按钮：

- 取消
- 保存权限配置

### 后端校验

- 必填字段不能为空。
- URL 基本合法。
- 同一岗位下同一权限不能重复绑定。
- 不物理删除权限项。
- 禁用权限项不清空历史 `permission_progress`。

### 新人端影响

- 新人端 `/permissions` 只展示启用权限项。
- 历史进度仍按 `permissionItemId` 读取，即使权限项已停用也不能崩。

## 12. 关键交互闭环

每个模块都必须满足：

1. 后台打开当前配置。
2. 编辑字段。
3. 点击保存。
4. 后端校验。
5. 数据写入数据库。
6. 页面 toast 成功。
7. 刷新后台，配置仍保留。
8. 新人端或复盘端读取到新配置。

模块对应下游：

- 岗位权限包：新人端 `/permissions`、`/permission-detail/:id`。
- D1 引导：新人端 `/d1`。
- 首周反馈配置：新人端 `/weekly-feedback`。
- 匿名反馈配置：新人端 `/anonymous-feedback`。
- 知识库管理：复盘页 `/review`。
- 匿名反馈池：复盘页 `/review`，但不进入管理者端匿名原文展示。

- 左侧导航、顶部栏、表格、编辑抽屉齐全。
- 所有保存接口走真实后端和数据库。
- 保存后刷新仍保留。
- 新人端/复盘端读取更新后的配置。
- 启停通过 `enabled` 或 `status`，不做物理删除。
- 后端校验必填、URL、重复绑定、父子关系、首周反馈不能全禁用。
- 审计字段写入 `updatedBy = demo-admin`。
- 反馈处理写入 `handlerName`、`handledAt`、`resolutionNote`。
- 管理者端不展示匿名反馈原文。
